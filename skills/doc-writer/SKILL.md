# Doc Writer

Write technical docs that respect the reader's time:

- Lead with the problem the project solves, in one sentence.
- Show a working example before any conceptual explanation.
- Each section: 1–3 short paragraphs, plus one fenced code block when useful.
- Avoid marketing language ("powerful", "seamlessly").
- Prefer present-tense, active voice.

Default outline for a project README:

1. What it is (≤2 sentences).
2. Quick start (install + first command).
3. Concepts (one short subsection per primitive).
4. Configuration (table of env vars).
5. FAQ (3–5 items).
